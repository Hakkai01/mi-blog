¿What is Git and GitHub?

24 de junio, 2026

A grandes razgos, y sin matarnos tanto la cabeza con las definiciones pues....

Git es un control de versiones, el cual llevara o lleva, qsyo, el registro de todas las modificaciones que le hemos realizado al codigo fuente del proyecto o a algun otro archivo perteneciente al mismo.

GitHub, por otro lado, actua como nuestro Google Drive (chimba de analogia), es el lugar en donde se encuentra el repositorio de nuestro proyecto, y con repositorio me refiero a todos los archivos pertenecientes al mismo (acostumbrense a la palabra repositorio, despues entenderan porque).

Dentro del GitHub, vamos a asignarle una URL en este caso a la pagina (cosa que ya hice, por algo estas leyendo esto), probamos como actuan las interfaces en diferentes dispositivos "Movil o PC", y demas cositas que explicare a continuacion.

Subir Proyectos a Github

25 de junio , 2026

Ya podemos empezar a mandar aca los comandos importantes:

"git init" -> Este comando crea un repositorio Git vacio dentro de la carpeta del proyecto, el archivo esta como no visible, asi que no pasa nada si no lo vemos.

"git branch -m main" -> Este comando renombra la rama "master" a main (creo que era por algo de que git ya no usa mas master y tiene que usar el main algo asi entendi).

"git add ." -> Con este comando le estamos diciendo a Git que registre todos los archivos que se encuentren en la carpeta del proyecto, y con todo me refiero a absolutamente todo.

"git commit -m "primer comit: bla bla"" -> El comit es como una foto, de lo que tenemos actualmente desarrollado, y el mensaje entre comillas sirve como descripcion de la foto, por lo general cuando hagamos algun cambio o modificacion siempre vamos a tener que colocar estos 3 comandos, desde el add, hasta el push, para guardar los avances que hicimos.

"git push -u origin main" -> Este es el comando que hace la magia, el que carga/sube el codigo, pero aca llegamos a la parte que me tire horas jodiendo por culpa de un token, resulta que la primera vez que haces un "push" de un nuevo proyecto, GitHub te pide "autenticacion", es decir te solicita usuario y contraseña para poder acceder a tu cuenta y completar la subida de archivos.

Token y Contraseñas

25 de junio , 2026

Resulta pasa sucede y acontece que, GitHub dejo de aceptar contraseñas normales por seguridad desde el 2021 (sopresa estamos en 2026), ahora utiliza TOKENS (no como los de las webcam.... bueno quizas si), Esto agrega un paso extra a la carga de archivos pero se hace una sola vez, (por proyecto)

Dentro de Github, tenemos que ir a "setting", dandole click a nuestra foto de perfil, y en ese apartado bajar en el menu del lateral izquiero hasta encontrar la opcion que dice "Developer settings" ahi se nos despliegan dos opciones mas, nosotros necesitamos la que dice "Tokens (Classic)", cuando entramos ahi nos aparece la opcion de generar un nuevo fokin TOKEN (classic obviamente).

En la parte de "note" colocamos el titulo del proyecto o como lo tengamos nombrado, y en el apartado de "expiration" le colocamos "no expiration", luego marcamos la casilla de "repo" verificando que todo el grupo aparezca marcado y por fin podremos darle a "Generate token".

Despues de eso nos va a mostrar algo como "JASFIUGQIbkjbaiubqfKSHAF.ghp" (se los juro que es asi) y por lo que mas quieran copienlo y no lo pierdan, porque esa es la contraseña para poder subir el archivo /proyecto, ahora si podemos ejecutar el "git push", llenamos nuestro usuario con el usuario de GitHub obviamente y en la parte de "Password" PONEMOS EL HIJUEPUTA TOKEN :3
