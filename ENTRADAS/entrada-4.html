Flexbox

4 de julio, 2026

Primero obvio para entrar con la definicion, el felxbox seria un modelo de CSS3 diseñado para organizar, alinear y distribuir elementos en una pagina web de forma flexible y responsiva.

Ahora, en terminos mas de la pagina, nosotros utilizamos dos elementos los cuales son "display:flex" y "justify-content", para poder alinear los elementos de forma horizontal dentro del header (donde tenemos el titulo del blog, inicio y sobre yo)

El display:flex actua como un contenedor, y dentro del contenedor, todos los elementos que coloquemos actuaran como sus hijos directos, dichos items se organizaran de forma horizontal automaticamente.



    "header"        ← contenedor flex
    "h1"..."/h1"    ← ítem 1
    "nav>..."/nav"  ← ítem 2
    "/header"

Sin el flex, tanto el h1 como el nav se apilarian uno encima del otro (segun le dicen flujo normal), pero con flex se alinean de forma horizontal.

Ahora el "justify-content" controla el espacio sobrante entre los item que conforman la fila dentro del contenedor.

Para entenderlo de mejor forma vuelvo y entro en los terminos del Eje de X y el Eje de Y, entonces, tenemos que tener 3 conceptos claros.

Justify-content: Organiza todos los elementos en base al eje de x, De izquierda a derecha.

align-items: Organiza todos los elementos en base al eje de y, De arriba a abajo.

flex-direction: Define la dirección en la que se alinean los elementos, tambien puede modificar el orden de los mismos.

Flexbox (otra vez) y Grid

12 de julio, 2026

Aver, antes de usar analogias para explicar como funciona el flexbox, tengo que definir como funciona el grid y en donde se podria implementar este mismo, aver ahora mi plan es cambiar todo el diseño de la interfaz, el como se organiza tanto la pestaña principal como los demas apartados por ende tengo que empezar a meter un poco mas el concepto de que es lo que viene siendo grid.

Para que sea un poco mas comprensible, flexbos solo puede trabajar en una direccion, es decir en una fila o una columna pero solo una cosa al tiempo en cambio grid trabaja dos dimensiones al mismo tiempo, filas y columnas por igual, por lo tanto es mucho mejor para los layout de paginas.

Ahora si, haciendo uso de una analogia, el flexbox funciona como organizar libros en un estante, segun la logica primero llenas la fila y despues la columna, pero con grid puedes hacerlo como si estuvieras metiendote dentro de un excel, jugando con filas y columnas en simultaneo.

De hecho con eso fue con lo que hicimos el apartado de "Sobre mi".

Media Queries

13 de julio, 2026

La logica detras de esto es que el media Queries es una forma de pensar no solo en que la pagina que estamos haciendo se vea bien en pc, sino que en cualquier dispositivo, hasta donde tengo claro el concepto, la mayoria de las paginas son responsive, es decir que se adaptan independientemente del ancho de la pantalla, para eso tenemos que jugar un poco con el css.

Las media queries le van a decir directamente al css, "si la pantalla es menor a tantos pixeles, ponga esto para que se vea mejor".


/* ===== RESPONSIVE ===== */
@media (max-width: 768px) {

    /* Las tres tarjetas del index pasan a una columna */
    .categorias {
        grid-template-columns: 1fr;
    }

    /* El layout de entradas pierde el sidebar */
    .entrada-completa {
        flex-direction: column;
    }

    .sidebar {
        position: static;
        width: 100%;
    }

    /* El perfil pasa a una columna */
    .perfil {
        grid-template-columns: 1fr;
    }

    .perfil-foto img {
        max-width: 280px;
        margin: 0 auto;
    }

    /* El grid de apuntes pasa a una columna */
    .entradas-grid {
        grid-template-columns: 1fr;
    }
}
